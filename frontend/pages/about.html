The Story

How It Came Together

WMS started as an academic capstone project with one goal: build something that actually works in the real world, not just on paper. The team of five students split responsibilities across backend development, frontend design, database administration, testing, and project management — mirroring how real engineering teams operate.

The backend was built using Django 5 and Django REST Framework, with SimpleJWT for token-based authentication and a PostgreSQL database hosted on Railway. Nine Django apps handle users, staff, aircraft, flights, gates, baggage, maintenance, incidents, and audit logging.

The frontend was rebuilt from React/JSX to plain HTML, CSS, and JavaScript for simplicity and speed — with glassmorphism design, dark theme, and responsive layouts throughout. The entire system is deployed on Render and accessible publicly.

The AI module uses scikit-learn linear regression trained on historical stock movement data to produce per-product demand forecasts and restock recommendations — a genuine ML feature, not a mock.
